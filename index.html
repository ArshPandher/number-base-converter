What is a Number Base?

A number base (also called radix) defines how many unique digits we use to represent numbers:

  • Decimal (Base-10): Uses digits 0-9. Our everyday number system.
  • Binary (Base-2): Uses digits 0-1. How computers store and process data.
  • Hexadecimal (Base-16): Uses digits 0-9 and letters A-F. Used for memory addresses and color codes.

Each position in a number represents a power of the base. For example:

  • In Decimal: 42 = (4 × 10¹) + (2 × 10⁰) = 40 + 2
  • In Binary: 101010 = (1 × 2⁵) + (0 × 2⁴) + (1 × 2³) + (0 × 2²) + (1 × 2¹) + (0 × 2⁰) = 32 + 8 + 2 = 42
  • In Hexadecimal: 2A = (2 × 16¹) + (10 × 16⁰) = 32 + 10 = 42

🖥️ Why Binary?

Computers use binary because they're built from transistors—tiny electronic switches that can only be in two states: ON (1) or OFF (0). This fundamental design makes binary the natural choice for computer architecture.

  • Transistors are reliable and efficient
  • Binary circuits are simple and fast
  • Error detection and correction is easier with 2 states than 10
  • All CPU instructions, memory, and data are ultimately stored as binary

🎨 Why Hexadecimal?

Hexadecimal is a human-friendly way to represent binary data. Since 16 = 2⁴, each hex digit represents exactly 4 binary digits (a "nibble").

  • Memory Addresses: A 32-bit address like 0xDEADBEEF is much easier to read than 11011110101011011011111011101111
  • Machine Code: Debuggers show assembly code using hex for readability
  • Color Codes: Web colors use hex: #FF5733 is easier than describing RGB values
  • Compact Representation: 8 hex digits = 32 bits (compact and human-readable)

💡 Key Takeaways for Computer Architecture

Digital Logic

All digital circuits operate on binary logic. Understanding binary is essential for computer architecture.

Data Representation

Numbers, characters, colors, and instructions are all represented using binary at the CPU level.

Debugging & Analysis

Hexadecimal is the standard notation in debuggers, assembly language, and memory inspection tools.

Performance

Understanding bit manipulation and number bases helps optimize code for performance.
